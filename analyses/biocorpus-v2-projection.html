biocorpus v2 — annotation-quality pass

Sequence markup removed, bookkeeping fields dropped, and a gate that keeps only records carrying real annotation that could in principle be learned from the sequence. This is what the full rebuild would produce, measured on samples of each source.

The headline. The quality bar removes UniRef50 in its entirety — 12,931,373 records, 79% of the corpus by tokens. UniRef ships a cluster name, a member count and a representative accession; none of that is an annotation of the sequence. What remains is small but almost entirely curated. See recovering scale.
Changes

What changed

  • Sequence markup removed. <protein>, <dna> and <rna> tags are gone — the anchor line already delimits the sequence FASTA-style, and the tags were syntax the model would have to learn around.
  • Bookkeeping dropped. Cluster size: 5 member sequence(s), Representative: A0ABP8NCJ6_9BACT, entry names and bare accession lists (PDB/EMBL/KEGG/RefSeq cross-references) are removed. They are identifiers, not biology, and nothing about them is predictable from a sequence.
  • Quality gate. A protein record is kept only if it carries at least one substantive annotation — function, catalytic activity, subcellular localisation, pathway, disease, GO terms, or per-residue features (domains, active/binding sites, PTMs, signal peptides, transmembrane spans, disulfides) — and is not an uncharacterised entry. Genomic records pass intrinsically: they carry coding regions, UTRs, exon structure, splice motifs, or cis-regulatory element type.

Measured

Recovering scale: TrEMBL is annotated where UniRef50 is not

The reason UniRef50 fails the gate is that its FASTA carries no annotation — but the underlying UniProt entries do. Measured on ~40,000 TrEMBL entries streamed from the flat file (not the API, whose ordering favours annotated entries):

TrEMBL entries with…fractionwhat it teaches
sequence features (domains, transmembrane, sites, disordered regions)83%domain boundaries, membrane topology, active/binding sites
GO terms63%molecular function / process / localisation
a FUNCTION or catalytic-activity statement~11%curated/rule-based function
≥1 substantive annotation (passes the gate)88%—

So TrEMBL (~149 M entries) holds on the order of tens of billions of tokens of annotated protein (88% × 149 M × ~407 tok ≈ ~50 B raw). The catch is redundancy — TrEMBL is un-deduplicated (that is why UniRef exists), so the right target is annotated + deduplicated:

  • Annotated UniRef50 (recommended). Join UniProt annotations onto the 38.8 M UniRef50 representatives (each has a RepID accession) and keep the annotated ones: ~88% × 38.8 M × ~450 tok ≈ ~15 B tokens — deduplicated and annotated, i.e. the original protein backbone, restored at the new quality bar. Cost: a 38.8 M-row join that requires processing the full TrEMBL flat file (118 GB gzipped).
  • Annotated TrEMBL + exact-sequence dedup (simpler). Skips the join but removes less redundancy than UniRef clustering; --stride can trim it to budget.
  • Multi-species genomics. Human dogma+splice is ~0.07 B; ~50 of the 356 Ensembl vertebrates the builder already handles offline would add a few billion (Regulatory Build is human/mouse only).
Caveat on this measurement. The 118 GB flat file is a single gzip stream — not randomly seekable — so this sample is the file head, which happens to be archaea-heavy (Haloterrigena, Natrinema, …; 1,078 organisms but a few proteomes dominate). The 88% is for that taxonomic slice; a representative figure needs a deeper or full pass. The qualitative gap is robust regardless: TrEMBL carries per-residue and GO annotation at 60–80%+, UniRef50's FASTA carries none.

Method

How these numbers were produced

Counts marked before are exact, from the built corpus. After counts apply the measured retention rate to those exact counts; token rates are re-measured with the Llama-3 / marin-community/marin-tokenizer tokenizer over the regenerated samples. Genomic record counts are unchanged because the gate passes them intrinsically. Nothing here has been run over the whole dataset yet — that is the next step once the rules settle.
